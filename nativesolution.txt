Ah, the classic WebView cookie problem! This is a common pain point when wrapping web apps in React Native. Let me break down some solutions:

The Problem
WebViews don't share cookies with the native app context, so OAuth flows break because:

Auth cookies set during login don't persist properly
Third-party cookie restrictions (especially on iOS)
Session state gets lost between WebView instances
Solutions to Consider
1. Token-Based Auth Bridge (Recommended)
Instead of relying on cookies, pass auth tokens between native and web:

// In React Native - inject token into WebView
<WebView
  source={{ uri: 'https://yoursite.com' }}
  injectedJavaScriptBeforeContentLoaded={`
    window.NATIVE_AUTH_TOKEN = '${userToken}';
    window.IS_NATIVE_APP = true;
  `}
  onMessage={(event) => {
    // Handle messages from web app (like logout, token refresh)
    const data = JSON.parse(event.nativeEvent.data);
    if (data.type === 'AUTH_TOKEN') {
      // Store token natively
    }
  }}
/>
Then in your Next.js app, check for the native token:

if (window.IS_NATIVE_APP && window.NATIVE_AUTH_TOKEN) {
  // Use token-based auth instead of cookies
}
2. Handle OAuth Natively
Do the KingsChat OAuth in React Native, then pass the session to WebView:

Use expo-auth-session or react-native-app-auth
Complete OAuth flow natively
Inject the resulting token into WebView
3. Custom URL Scheme for Auth Callback
Register a deep link (e.g., lwsrh://auth-callback)
OAuth redirects to your deep link
Native app catches it, extracts token, injects into WebView
4. SharedCookies (iOS) + Third-Party Cookies (Android)
<WebView
  sharedCookiesEnabled={true}  // iOS
  thirdPartyCookiesEnabled={true}  // Android
/>
This helps but isn't bulletproof.

My Recommendation
Go with Option 1 + 2 hybrid:

Detect if running in native app (window.IS_NATIVE_APP)
If native, handle OAuth in React Native
Pass token to WebView via injectedJavaScript
Your Next.js app checks for native token first, falls back to cookies for PWA
